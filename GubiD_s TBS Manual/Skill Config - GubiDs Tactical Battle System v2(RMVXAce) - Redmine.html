Skill Config¶

Put these in the notebox of a skill.

Skill range¶

range=[range_max, Field, LINE_SKILL?, exclude_caster?, range_min, vertical_range, vertical_aoe]

range_max, range_min: numerical; determines the maximum and minimum ranges within which the skill can be targeted. 0 is the tile the caster is standing on.
Field: determines the AoE of the skill; 0 would affect only the target tile, whilst 1 would affect directly-adjacent tiles in addition to the target.
LINE_SKILL?: true or false; determines whether the caster can only target tiles in a horizontal or vertical line from themselves. AoE skills can still affect units outside of the line, however.
exclude_caster?: true or false; prevents the caster from being affected by the skill, even if they're within the AoE.
vertical_range: integer; optional, the number of pixels upwards or downwards from the source tile that the skill will accept as a valid target tile. Only used by MGC's isometric battle engine, and only if range_max is more than 1.
vertical_aoe: integer; optional, the number of pixels upwards or downwards from the target tile that the skill will apply AoEs to. Only used by MGC's isometric battle engine, and only if Field is 1 or more.

AI skill rating¶

ai_rating=rating

rating: numerical; Defines the rating applied to the skill by an AI-controlled Actor. The higher the value, the more likely the Actor is to use the skill over other skills or actions. The default rating of any skill without this defined is 5. AI-controlled enemy units do not respect this tag, and use the rating of the skill as set normally in the database.

Use Action¶

use_action=value

value: true/false; Defines whether this skill should count as an 'action' while in battle. If true (default) then it will be the only action that can be performed in a turn. This was implemented to provide a 'buff skill' option so that one could buff their character before delivering their attacks. There is a max number however of 'mini actions' that you can perform. See: Max_Turn_Skill_Usage in the Skills section.

Skill Wait¶

skill_wait=[Cast_time, ATB_recover, End_Turn]

Cast_time: numerical; ?
Units of AT required to cast the given skill

ATB_recover: numerical; ?
Upon casting, recover the % of AT units until next turn.

End_Turn: True or False; ?
This option forces the players turn to end upon the placement of this skill. (Prevents the ability to move after skill placement)

Summon tags¶

Both of these cause a skill to summon the specified actor or enemy.
Probably requires a bit more setup in the scripts, as currently the IDs of summon classes point to non-existent classes.

actor=actor_id
enemy=enemy_id

Projectiles Deprecated - Utilize Action_Config¶

Both of these tags cause the use of the skill to fire a projectile at the target area prior to the skill's actual effects triggering; you need to save an image of the desired projectile with the same name as the skill in Graphics/Pictures/GTBS/Projectiles for this to function properly. 'Normal' projectiles travel straight, whilst 'arched' projectiles are thrown up in an arc.

projectile=normal
projectile=arched

Secret Hunt¶

secret_hunt

When present, this tag allows you to 'poach' enemies as they are killed. This only works when you deliver the final blow to the enemy. For instructions to config the enemy for secret_hunt, please see the Enemy Section.

Capture¶

capture_enemy

When present, this tag allows you to 'charm' enemies to join your team. This will be in effect each time the attack is delivered. For instructions on how to configure your enemies for capture, please see the Enemy Section.

Chain Lightning¶

chain_lightning

When present, this tag triggers enemies targeted to reduce dmg each time it is delivered. The damage reduction is done in percentages as per the CHAIN_LIGHTNING_CURVE found in the Skill Setup section. The first target is assumed 100% damage, and all subsequent will follow the curve. If there are more enemies than there are curve entries. Then the final enemies will damage at the rate of the last curve entry.

Weapon Effects and Actor Movement Animation Deprecated - Utilize Action_Config¶

anime=ANIME_NAME

ANIME_NAME: string; This option allows you to to define custom weapon animation and actor movement animations. For example, in the demo provided, there is a default ANIME called, REG_SWING. This function MUST be manually configured in the ANIME_ACTION_SEQUENCE hash found inside the Skill section. For more information in regards to setting this extensible feature up, please see the Skill Section.

Animations¶

play_anims=NUM, NUM, NUM

NUM: number; This function allows you to override the animations that should be played. This is intended to be used in conjunction with the anime= tag. You can use this tag to display 'user' animations followed by 'enemy' animations. If this item is not defined in a skill. Then
the default animation (database field) will be used automatically.

Direction Flags¶

Note Entries:

ignore_dir

Utilizing this flag causes directional damage adjustment to be ignored. Thus all damage will be inflicted at the 100% rate (supposing you hit)

back_dmg_rate = NUM
side_dmg_rate = NUM
front_dmg_rate = NUM

NUM: number; This function allows you to define the damage rate for the given direction on the client.

AT Adjustment¶

reduce_at = NUM

NUM: number; This function allows you to reduce the defenders AT by the given percentage, thus delaying their turn. (Applicable to AT mode only)

Damage Options¶

prevent_friendly_dmg
friendly_only

The existence of these note tag entries will either prevent friendly targeting or ensure it. This is required to achieve skill of such that cannot target enemies or only target when in 'ATTACK_ALLIES=true' mode.

Usage Flags¶

reset_action
reset_move

The existence of these note tag entries, causes the targeted to gain another action phase, movement phase, or both.
