Action Config¶

Movement¶

Usage:   move [T]: [OP]

[T]: Target Type
  user/self (implies the user of this action)
  target/enemy (implies the target of this action)
  picture/pic/image/img/icon/ico (implies to move an image that is on the scene)
[OP]: Optional Params
  Params for both Battler and Image Types:
    absolute|abs|xy NUMBER NUMBER
      Moves [T] to X,Y coordinates on the screen.  Be they on or off the valid area. 

    time|t|duration|dur NUMBER
      The duration in which this movement take in time to complete.

    away|back|backward NUMBER
      Move the image|battler [T] AWAY from target (be that attacker or defender) the given amount of pixels specified.

    forward|approach|toward
      Move the image|battler [T] TOWARD the target (be that attacker or defender) the given amount of pixels specified.

    target|enemy [LOCATION]
      Moves [T] to opposing attacker/defender
      [LOCATION] - OPTIONAL
        Refers to target body location.  Valid values are:
        feet/bottom, head/top, body/middle, hand1|arm1|right, hand2|arm2|left
      NOTE: body is assumed if not provided. 
      NOTE: If any variant regarding hands is used(only valid with images), it will automatically set the filename to that of the weapon in that hand for the given target.  It also automatically selects the center technique for rotating that given weapon. 

    user|self [LOCATION]
      Moves [T] to opposing attacker/defender
      [LOCATION] - OPTIONAL
        Refers to target body location.  Valid values are:
        feet/bottom, head/top, body/middle, hand1|arm1|right, hand2|arm2|left
      NOTE: body is assumed if not provided. 
      NOTE: If any variant regarding hands is used(only valid with images), it will automatically set the filename to that of the weapon in that hand for the given target.  It also automatically selects the center technique for rotating that given weapon. 

    wait|halt|complete
      Wait for the current movement to complete before proceeding with the next action.

  Params specifically for BATTLER types: (move user: | move target: )
    right|left|up|down NUMBER
      The adjust the image|battler to the right|left|up or down of the target point (be that absolute or battler etc) the given amount of pixels specified.

    return
      Returns the battler to their starting position.

  Params specifically for Images:
    index|idx NUMBER
      The icon index to be displayed.

    file|filename|src|source TEXT
      The image filename to be loaded from the pictures folder

    wep|weapon NUMBER
      Defines the actors weapon in the given hand.  1 and 2 is generally all this will have set, but it has the capability for more to be provided given there are more weapons. 

    opacity|op NUMBER
      Sets the opacity of the object to the specified value.  If it is at another value, it will transition  to the new opacity in the given amount of time in which the action occurs.

    attack|strike|hit|fire
      Used in conjunction with a wep|weapon tag to play the weapon action (equivalent to calling "animation user: wep 1").  

    key|tag TEXT
      Is the unique identifier to this image.  This is present so that you can spawn multiple of the same image onto the scene using alternate keys to manage each ones data.

    origin|center|cnt [ at] *
      Valid types: 
        1-9 - Numpad direction indicating where the origin point should be. 
        X Y value - The X,Y coordinates on the image that should act as the center(rotating point).
        LR|LL|UR|UL - Define the corner in which the image rotation point should be. 
      If not specified it is assumed UL just like all normal images.

    mirror|flip true|false
      indicates weather to mirror the given image or not.  This will only occur on image creation.

    angle NUMBER [OPTIONAL]
      The angle in which the given image should rotate to.  
        The optional param on this tag is '<>'.  As in just <>.   This causes the image to MIRROR the given angle when facing right.  This is helpful for side view animations.
      NOTE: If the optional param of <> is not supplied, it is assumed that this animation will be relative to the TARGET direction if supplied for this image object. 

    zoom|scale x|y NUMBER
      Zoom the given axis of x or y to the given amount.  The value is divided by 100 to make a percent, so negative values will not be accepted. 1% would be 1 and 400% would be 400.  There is no cap on this. 

    z NUMBER
      Sets the Z value of the given image to the specified value.  This is helpful when needing to layer various images to achieve an affect.
