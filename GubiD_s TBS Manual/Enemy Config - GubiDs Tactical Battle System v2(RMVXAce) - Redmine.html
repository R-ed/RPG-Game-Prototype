Enemy Config¶

Put these in the notebox of an enemy.

Movement range¶

move=range

range: numerical; the maximum number of tiles from its current location that an enemy can possibly move in a turn.

View range¶

view=range

range: numerical; the maximum number of tiles from its current location that an enemy can see player-allied units. OUtside of this range, behaviour differs depending on a couple of AI settings.

Enemy class¶

class=class_name

class_name: text; the name of the class the enemy displays during battle. Purely cosmetic.

Enemy level¶

level=value

value: numerical; the level of the monster. Monster level affects experience gained from actions featuring this monster.

Enemy state add¶

Applies the specified state to the enemy's attack. The game will attempt to apply this state to any targets of the attack.

state_add=state_id

state_id: numerical; ID of the desired status effect in the database.

Enemy size¶

large=size

size: numerical; enemy will take up size^2 tiles, and be unable to pass through smaller spaces.

Enemy weapon range¶

w_range=[Far_range, min_range, is_bow?, line_to_target?, AoE, vertical_range, vertical_aoe]

Far_range, min_range: numerical; the maximum and minimum ranges in which an attack can be targeted, by this enemy. 0 specifies the tile the unit is standing on.
is_bow?: true or false; if true, allows the weapon to target diagonal tiles within the minimum and maximum range, as long as BOW_LSHAPE in General Settings is true.
line_to_target?: true or false; if true, attacks with this weapon affect all possible targets between the attacker and the target tile.
AoE: numerical; determines the AoE of the skill; 0 would affect only the target tile, whilst 1 would affect directly-adjacent tiles in addition to the target.
projectile?: true or false; doesn't actually determine whether projectiles are used or not, as the tag below now determines it, but the note tag checker still wants 'true' or 'false' here.
vertical_range: integer; optional, the number of pixels upwards or downwards from the source tile that the attack will accept as a valid target tile. Only used by MGC's isometric battle engine, and only if
far_range is more than 1.
vertical_aoe: integer; optional, the number of pixels upwards or downwards from the target tile that the attack will apply AoEs to. Only used by MGC's isometric battle engine, and only if AoE is 1 or more.

Projectiles¶

Both of these tags cause the enemy to fire a projectile when attacking; you need to save an image of the desired projectile with the same name as the enemy in Graphics/Pictures/GTBS/Projectiles for this to function properly. 'Normal' projectiles travel straight, whilst 'arched' projectiles are thrown up in an arc.

projectile=normal
projectile=arched

Enemy hidden HP/MP¶

unknown_hp

Just putting that tag into an enemy's notebox causes the enemy's HP and MP values to be hidden.

Enemy Face¶

face_name=FACE_FILE
face_index=INDEX

FACE_FILE: string; Used to determine the filename which contains the face graphics to be used.
INDEX: number; Used to determine which face within the file should be used. This works the same way as the rpg maker core face system.

Enemy AI tactic¶

tactic=tactic_name

tactic_name: text; name of a specific tactic defined in the AI Setup - valid tactics in the default project are 'attack', 'defend', 'assassin', 'healer', 'berseker', 'team' (and 'default'?)

Enemy Description¶

description=VALUE

VALUE: string; Used to determine description for enemy when the status screen is displayed. Will only go to the first line break.

Secret Hunt Results¶

hunt_results=[ [TYPE, ID, RATE] ]
hunt_results=[[TYPE, ID, RATE],[TYPE, ID, RATE]]
hunt_results=[[TYPE, ID, RATE],[TYPE, ID, RATE],[TYPE, ID, RATE]]

TYPE, ID, RATE: Number;
TYPE represents the item type that should be awarded. Valid entries for this item are:
  • 0: Item
  • 1: Weapon
  • 2: Armor

ID represents the ID of the item within the specified TYPE to give.

Rate represents the rate at which this item is given. 

Note that only 1 item is given.  Also that the array is processed backwards.  The recommend way for setup is [[Common][LessCommon][Rare]]

Summons¶

summon

When this tag is present it identifies this enemy as a Summon type. Which means when they die they will be removed from the battlefield regardless of option selected.

Capture Settings¶

actor=ID
cap%=NUMBER
cap_mult=NUMBER

actor= and cap% are required for this enemy to be 'capturable'. actor= determines which actor ID which should be added to the party should this enemy be successfully 'captured'
cap%= is the rate at which these enemy is captured.
cap_mult= is the multiplier. When set allows the enemy to be captured more easily when near death vs full life. If not provided, this value is assumed 1.

Death Animation¶

death_anim=ID

ID: number; Used to determine the Animation ID that should be shown when this enemy dies.

Wander Settings¶

prevent_wander

When ALLOW_AI_WANDER is true, this tag will prevent this particular enemy from randomly roaming the battlefield when there is no 'enemy' units in sight.

Weapons¶

weapons=ID, ID, ID, etc

ID: number; Used to determine the Weapon ID that should be used for this enemy should they deliver an attack associated with an weapon animation.
Only the first weapon will be used by default. However, if the enemies ANIME_ACTION has the keyword 'NEXT_WEAPON' then each additional weapon will be used in order specified.

Traverse Type:¶

traverse=TYPE

TYPE: number; Used to determine which type of terrain tags affect this unit. Valid values are:
  • 0 : Normal (Affected normally by all terrains)(default if not defined)
  • 1 : Thick (Not affected by thick terrain (assumed Terrain Tag 1))
  • 2 : Rocky (Not affected by mountainous(rocky) terrain. (Assumed Terrain Tag 2))
  • 3 : Mounted (Not affected by thick or rocky terrain)
