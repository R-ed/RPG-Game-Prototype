General Config¶

This section will cover 'Engine Settings' that are used to overall affect the behaviour of the GTBS system.

Boolean (TRUE/FALSE) values¶

Option Name Default Value Description
STOPPED_ANIM true This allows characters to 'walk in place' on the map while idle when true
ACTOR_PLACE_USES_INDEX true This setting determines whether to use PartyIndex vs DatabaseID for ACTOR# events at battle start.
When the value is TRUE then the ACTOR1 event indicates that the PARTY LEADER should be placed on that tile during battle start.
When the value is FALSE then the ACTOR1 event indeicates that the Database Actor(ID of 1) should be placed on that tile during battle start.
Place events will still only be available for 'party' members.
BTEST_TBS true This option is so that you can test a battle using the TBS right from the database editor.
If TRUE then when battle test is initiated, the TBS will be used to test it.
If FALSE then the default battle system (or appropriate battle script) will be used instead.
FORCE_MOVE_THEN_ACT false This option makes so that if you do not move before you act, you lose your chance to move.
If TRUE then if an action will force the end of your turn. *1 There are exceptions to this
If FALSE then you can attack/move in any order desired.
*1 An option was added to prevent the usage of your 'action' for certain skills. If enabled and configured those 'skills' will not prevent your move from occurring.
AUTO_DEFEND false This option makes the actor/enemy automatically defend if they have not used their 'action' but end their turn.
If TRUE then:
The menu word 'WAIT' will be 'DEFEND' (from Vocab) when an action has not been performed yet.
The 'GUARD' skill will be applied as the actor/enemies turn as they complete it. (If they have not acted)
FACE_ENEMY true This option allows the placed actors to face their closest enemy ( could be used for a cool ambush scene ) when the battle has begun.
If TRUE:
At the start of battle. All actors/enemies will face their closest enemy target.
If False:
All units will face DOWN until acted upon.
ATTACK_ALLIES true Allows Player and AI to target their friendly units
TEAM_THROUGH true Set ability for characters to walk through other team member squares
POP_DAMAGE_IND true When more than a single target is damaged, then the animation and damage value is shown completely separate
SHOW_ITEMGOLD_IMMEDIATELY false Allows you to gain gold/exp/dropped items immediately on the death of a target
REVIVE_ANY_DEAD false If TRUE all 'revive' skills will call a window for you to selected from dead members. It will revive the character to the selected tile.
if FALSE, then you must target the dead body with your skill. This option is used in conjunction with REMOVE_DEAD feature
USE_WAIT_SKILLS true Allows 'casting time' for skills. Once elapsed then the skill will take effect.
ALLOW_SKILL_TARGET true This option is only used when USE_WAIT_SKILLS is true.
When targeting, you have the option to target the tile (x,y position) or the target (if available). Which may allow the skill to be performed outside of the normal castable area.
EXP GAIN OPTIONS See options in Experience page
BOW_LSHAPE true Allows diagonal attacks for weapons/enemies marked as BOWS
COUNTER_ALL false Forces a automatic counter attack to all effects on a battler. Think Tactics Ogre.
COUNTER_TEAM false Only Applicable when ATTACK_ALLIES = true.
Allows ally units to retaliate to attacks they may receive from you.
USE_ACTION_POINTS
ENCOUNTER_MOVING_METHOD true Prevents a player from passing by an enemy without being forced to stop. For more information see Encounter Move Positioning

Integer(Numeric) values¶

Option Name Default Value Description
MODE 0 0-Active Time
1-Team Mode
This function allows you to change whether AT (action timing) is used to determine turn order or all members of a team should act in a single 'turn'
REDUCE_AT_PERC 5 Reduces the DEFENDER AT value in percent. This will provide a tactical advantage to surround and attack an enemy as they will be unable to act as frequently.
BTEST_ID 0 The MapID that should be called when Battle Testing from the database
Status_Gauge_Font_Size 16 Default font size used for NAME on the status dialog
Recover_After_Battle 0 0 - Never Recover
1 - Neutrals Recover
2 - All Units Recover
VIC_COM 2 The COMMON_EVENT that should be played upon a VICTORY. This is played at the end of EVERY battle
FAIL_COM 1 The COMMON_EVENT that should be played upon a LOSS. This is played at the end of EVERY battle that is lost. This usually only contains a 'Game_over' call.
There are exceptions to this being played. Specifically when 'can_lose' is not marked on a planned battle.
REMOVE_DEAD 2 0 = Do NOT remove any dead from the battlefield
1 = Remove Enemies Only from the battlefield
2 = Remove ALL dead from the battlefield
