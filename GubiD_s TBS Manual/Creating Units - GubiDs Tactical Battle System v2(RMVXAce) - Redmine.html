Creating Units¶

Mostly units are created as they would normally be created in the database, but a few extra steps need to be taken to produce usable units in GTBS.

Actors¶

Actor and Class Setup
Actor Config

Graphics¶

An actor will have two to three graphics. A required out of battle graphic (Graphics/Characters), a required in battle graphic (Graphics/Battlers), and potential for a Mini-Scene graphic (Graphics/Battlers). The same name is required for the three graphics with the Mini-Scene graphic having '_MINI' at the end of it.

Example:
Out of Battle: Graphics/Characters/Actor3.png
In Battle : Graphics/Battlers/Actor3.png
Mini Scene : Graphics/Battlers/Actor3_MINI.png

The character in the same position in the Battlers graphic will automatically be used for the Actor in GTBS. If an actor is not given a Mini Scene sprite then a 'no_mini' tag must be added to the actor's notebox for the actor to show up in the Mini Scene.

Graphics¶

Enemy units need two graphics in the Battlers folder; a regular graphic used in the database and the default VX Ace battle system, and a single-character tileset (one 3x4 set of sprites for a character) with the same name as the regular graphic prefixed by $, used for the unit in the tactical battle system for both movement and attack animations.
EG. 'cat', a picture of a cat, and '$cat', a single 3x4 character set, both need to be in the Battlers folder.
